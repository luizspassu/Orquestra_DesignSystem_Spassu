⚠️ Regra Crítica de Acessibilidade

O token #FA4616 (orange) aplicado como cor de texto em tamanhos menores que 18px tem razão de contraste inferior a 3:1 sobre fundo branco — reprovado por WCAG AA. Sempre substituir por #C2410C (orange-text), que possui razão ≥ 4.5:1 e é visualmente semelhante.

Violações a Corrigir

Problemas identificados no protótipo atual que devem ser corrigidos antes da entrega ao dev. Ordenados por severidade.

🔴 CRÍTICO — Cor de texto: #FA4616 sobre fundo branco
O token #FA4616 está sendo usado diretamente como color em textos de KPI cards (valores financeiros) e possivelmente em outros componentes. Esta cor tem razão de contraste de ~3.0:1 sobre fundo branco, reprovando WCAG AA (mínimo 4.5:1 para texto normal).

Ação: Substituir textColor: C.orange por textColor: "#C2410C" em todos os KPI cards e em qualquer texto colorido de destaque.
// ❌ Antes (violação) { label:"Planejado", textColor: C.orange } // #FA4616 — contraste 3.0:1 // ✅ Depois (correto) { label:"Planejado", textColor: "#C2410C" } // contraste 4.5:1 ✓
🔴 CRÍTICO — Touch target de ícones: 26×26px (mínimo: 36px)
Botões de ícone (fechar sidebar, editar, excluir, link) estão com minWidth/minHeight: 26px ou sem dimensões mínimas definidas. Isso prejudica usabilidade em telas touch e viola WCAG 2.5.5 (AA).

Ação: Atualizar todos os botões de ícone para minWidth: 36px; minHeight: 36px. Manter o padding ajustado para não aumentar desnecessariamente o espaço visual.
// ❌ Antes style={{ minWidth:26, minHeight:26, padding:4 }} // ✅ Depois style={{ minWidth:36, minHeight:36, padding:8 }}
🟡 MÉDIO — orquestra_preview.html desatualizado
O arquivo orquestra_preview.html é um snapshot da Fase B e não inclui as mudanças da Fase C (aba Financeiro, dados de mock atualizados, FinanceiroView, 5 fases no MOCK_DETAIL). O protótipo real está em ContractDetailPage_preview.jsx.

Ação: Regenerar o HTML de preview a partir do JSX atual para que o time possa abrir localmente sem Node.js.
🟡 MÉDIO — Token C.orange ausente no arquivo para textColor
O objeto C não possui um token orangeText (que seria #C2410C). Isso obriga os devs a usar o hex hardcoded, quebrando a consistência do sistema de tokens.

Ação: Adicionar orangeText: "#C2410C" ao objeto C e usar C.orangeText em todos os textos de destaque.
// ✅ Adicionar ao objeto C const C = { ...tokens existentes, orangeText: "#C2410C", // substituto seguro para texto };
